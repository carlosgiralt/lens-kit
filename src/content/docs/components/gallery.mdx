`lk-gallery` is a modal lightbox that wraps any trigger elements — images, videos, or links — and opens them in a full-screen modal with carousel navigation and pan/zoom support powered by `lk-panzoom`.

## Basic Usage

Wrap `<img>` elements with `lk-gallery`. Clicking any image opens the modal at that index.

```html
<lk-gallery layout="grid">
  <img src="https://picsum.photos/id/1018/1920/1080" data-caption="Beautiful mountains" />
  <img src="https://picsum.photos/id/1015/1920/1080" data-caption="River landscape" />
  <img src="https://picsum.photos/id/1043/1920/1080" data-caption="Forest path" />
</lk-gallery>
```

## Gallery with Carousel

Nest a `lk-carousel` inside `lk-gallery`. The gallery auto-detects all `<img>` elements and maps clicks to the correct modal slide.

```html

    <img src="https://picsum.photos/id/1043/1920/1080" />
  </lk-carousel>
</lk-gallery>
```

## Customizable Toolbar

Control which actions appear in the modal header using the `toolbar` attribute. Pass a comma-separated list of action names.

| Token | Description |
|---|---|
| `zoom` | Zoom in button (alias for `zoom-in`) |
| `zoom-in` | Zoom in |
| `zoom-out` | Zoom out |
| `1:1` | Toggle 1:1 / 2× pixel view |
| `rotate-left` | Rotate 90° counter-clockwise |
| `rotate-right` | Rotate 90° clockwise |
| `flip-h` | Flip horizontally |
| `flip-v` | Flip vertically |
| `reset` | Reset zoom, rotation, and flips |
| `\|` | Visual divider |
| `play` | Start / stop slideshow (3 s interval) |
| `fullscreen` | Toggle fullscreen |
| `close` | Close button |

```html
<lk-gallery toolbar="zoom-in, zoom-out, 1:1, rotate-left, rotate-right, flip-h, flip-v, reset, |, play, fullscreen, close">
  <img src="https://picsum.photos/id/1018/1920/1080" />
  <img src="https://picsum.photos/id/1015/1920/1080" />
</lk-gallery>
```

<DemoGallery toolbar="zoom-in, zoom-out, 1:1, rotate-left, rotate-right, flip-h, flip-v, reset, |, play, fullscreen, close">

## Custom Toolbar Actions

Register additional toolbar buttons via `GalleryToolbar.registerTool(name, renderer)`. The renderer receives a live context object and returns a Lit `html` template — the button is rendered inline with the built-in tools and fires the same `action` CustomEvent.

The example below adds a **Download** button that saves the currently visible image.

```js
import { GalleryToolbar } from 'lens-kit';
import { html } from 'lit';

const iconDownload = html`
  <svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" viewBox="0 0 24 24"
       fill="none" stroke="currentColor" stroke-width="2"
       stroke-linecap="round" stroke-linejoin="round">
    <path d="M21 15v4a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2v-4"/>
    <polyline points="7 10 12 15 17 10"/>
    <line x1="12" y1="15" x2="12" y2="3"/>
  </svg>
`;

// Call once before the component renders (module scope is fine)
GalleryToolbar.registerTool('download', (ctx) => html`
  <button
    part="action-button action-download"
    class="action-btn"
    aria-label="Download image"
    @click=${() => ctx.dispatch('download', { index: ctx.currentIndex })}
  >${iconDownload}</button>
`);
```

Use the registered name in the `toolbar` attribute like any built-in token:

```html
<lk-gallery toolbar="download, |, zoom, play, fullscreen, close">
  <img src="photo-1.jpg" />
  <img src="photo-2.jpg" />
</lk-gallery>
```

Listen for the `action` event on `lk-gallery` to handle it:

```js
const gallery = document.querySelector('lk-gallery');

gallery.addEventListener('action', (e) => {
  const { action, index } = e.detail;
  if (action !== 'download') return;

  const src = gallery.items[index]?.src;
  if (!src) return;

  const a = document.createElement('a');
  a.href = src;
  a.download = src.split('/').pop() ?? 'image';
  a.click();
});
```

### Without Lit (plain DOM API)

If your project doesn't use Lit, call `GalleryToolbar.registerToolElement` instead. Pass a factory that creates and returns a plain `HTMLElement` — no tagged-template syntax required.

```js
import { GalleryToolbar } from 'lens-kit';

GalleryToolbar.registerToolElement('download', (ctx) => {
  const btn = document.createElement('button');

  // Apply the same CSS class used by built-in buttons
  btn.className = 'action-btn';
  btn.setAttribute('aria-label', 'Download image');

  // Use CSS ::part() for targeted styling: part="action-button action-download"
  btn.part.add('action-button', 'action-download');

  btn.innerHTML = `
    <svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" viewBox="0 0 24 24"
         fill="none" stroke="currentColor" stroke-width="2"
         stroke-linecap="round" stroke-linejoin="round" aria-hidden="true">
      <path d="M21 15v4a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2v-4"/>
      <polyline points="7 10 12 15 17 10"/>
      <line x1="12" y1="15" x2="12" y2="3"/>
    </svg>
  `;

  // ctx.dispatch fires the same "action" event lk-gallery already listens to
  btn.addEventListener('click', () =>
    ctx.dispatch('download', { index: ctx.currentIndex })
  );

  return btn;
});
```

The event handling side is identical — listen for `action` on `lk-gallery`:

```js
document.querySelector('lk-gallery').addEventListener('action', (e) => {
  if (e.detail.action !== 'download') return;
  const src = e.target.items[e.detail.index]?.src;
  if (!src) return;
  Object.assign(document.createElement('a'), {
    href: src,
    download: src.split('/').pop() ?? 'image',
  }).click();
});
```

:::note
The factory is called on every render cycle (e.g., when the slide changes or play state toggles). Event listeners attached inside the factory are re-registered on each call — this is intentional and has no meaningful overhead for toolbar buttons.
:::

### Renderer context

The second argument to `registerTool` receives a `CustomToolContext` object:

| Field | Type | Description |
|---|---|---|
| `host` | `GalleryToolbar` | Reference to the toolbar element |
| `currentIndex` | `number` | 0-based index of the current slide |
| `total` | `number` | Total number of slides |
| `isPlaying` | `boolean` | Whether slideshow auto-play is active |
| `isFullscreen` | `boolean` | Whether the gallery is in fullscreen mode |
| `isHtmlSlide` | `boolean` | Whether the current slide is HTML/video — custom tools are **never** auto-hidden by this flag, unlike built-in panzoom tools |
| `dispatch(action, detail?)` | `function` | Fires the standard `action` event; `detail` is merged into `event.detail` |

## Native Video Support

`<video>` elements are first-class citizens. When clicked, the player is duplicated into the modal so it scales correctly with pan/zoom.

```html
<lk-gallery toolbar="fullscreen, close" show-thumbnails>
  <video width="100%" height="auto" poster="https://storage.googleapis.com/gtv-videos-bucket/sample/images/BigBuckBunny.jpg" style="max-width: 400px; cursor: pointer;">
    <source src="https://storage.googleapis.com/gtv-videos-bucket/sample/BigBuckBunny.mp4" type="video/mp4" />
  </video>
</lk-gallery>
```

## Trigger Elements

The gallery scans its slot for these element types:

| Element | Source attribute |
|---|---|
| `<img>` | `src` or `data-src` |
| `<a>` | `href` |
| `<video>` | cloned into modal |
| Any element | `data-src` |

## Data Attributes

These attributes can be placed on any trigger element.

| Attribute | Description |
|---|---|
| `data-caption` | Caption text displayed at the bottom of the modal slide. |
| `data-src` | Override or provide the media source for elements that do not natively have `src`. |
| `data-type` | Set to `"image"` or `"html"` to tell the gallery how to render the item. |

## Programmatic Control

```js
const gallery = document.querySelector('lk-gallery');

// Open at a specific index
gallery.initialIndex = 2;
gallery.open = true;

// Close programmatically
gallery.close();

// Listen for close events
gallery.addEventListener('close', () => console.log('Gallery closed'));
```

## Keyboard Navigation

When the modal is open the following keyboard shortcuts are active:

| Key | Action |
|---|---|
| `←` Arrow Left | Previous slide |
| `→` Arrow Right | Next slide |
| `Esc` | Close the modal |

## Properties

| Attribute | Type | Default | Description |
|---|---|---|---|
| `layout` | `'inline' \| 'grid'` | `'inline'` | Set to `'grid'` to render a CSS grid thumbnail view of the trigger elements. |
| `toolbar` | `string` | `'zoom, play, fullscreen, close'` | Comma-separated list of toolbar actions. |
| `show-thumbnails` | `boolean` | `false` | Show a thumbnail strip at the bottom of the modal. |
| `open` | `boolean` | `false` | Open or close the modal programmatically. |
| `initial-index` | `number` | `0` | Index of the slide to open when the modal is first shown. |

## Django / Python Booleans

`lk-gallery` also coerces string attribute values so Python-style booleans work in templates.

- truthy: `"True"`, `"true"`, `"1"`, empty attribute presence (`show-thumbnails`)
- falsy: `"False"`, `"false"`, `"0"`, `"off"`, `"no"`

Use kebab-case attribute names in markup (`show-thumbnails`, not `showThumbnails`).

```django
<lk-gallery
  show-thumbnails="{{ show_thumbnails }}"
  open="{{ open_gallery }}"
>
  ...
</lk-gallery>
```

## Events

| Event | Description |
|---|---|
| `close` | Fired when the modal closes, whether by user action or `gallery.close()`. |

## CSS Custom Properties

| Property | Default | Description |
|---|---|---|
| `--gallery-cols` | `auto-fit` | Grid column count (grid layout only). |
| `--gallery-min-width` | `150px` | Minimum column width in grid layout. |
| `--gallery-gap` | `16px` | Gap between grid items. |
| `--gallery-image-height` | `150px` | Height of each grid thumbnail. |
| `--gallery-image-fit` | `cover` | `object-fit` for grid thumbnails. |
| `--gallery-radius` | `8px` | Border radius for grid thumbnails. |

```css
lk-gallery {
  --gallery-cols: 4;
  --gallery-gap: 8px;
  --gallery-image-height: 180px;
  --gallery-radius: 4px;
}
```
